组会汇报

2026 年 3 月

动机与研究问题

基于 LLM 的 AI Agent（如 The AI Scientist）已能自主跑实验，但面对复杂任务时频繁遇到上下文窗口瓶颈。

核心洞察：科研 = 迭代试错。每次实验产生的历史信息会不断积累，逐步压缩 Agent 做下一次实验的可用空间。

核心问题：给定上下文窗口 \(C\)，AI Agent
能自主解决哪些交通工程研究问题？
其可解性边界能否被理论预测？

研究假设

H1 Agent 能执行的连续实验次数存在可计算上界 \(d^*\)

H2 不同问题有不同 \(C_{\min}\)，据此可分类"可解/不可解"

H3 压缩比 \(\alpha\)（理想 vs 实际摘要）是限制深度的主导因素

研究范围：仅关注自主实验阶段（不含 idea 生成和论文撰写）

核心思想：上下文约束下的研究树搜索

CCTS 形式化模型

将 AI 自主实验建模为上下文约束下的树搜索：每个节点 = 一次实验，可用上下文随深度递减。

上下文分配

\[ C = \underbrace{c_{\text{sys}} + c_{\text{prob}}}_{S:\ \text{Static}} + \underbrace{c_{\text{hist}}(v)}_{\text{Inherited}} + \underbrace{c_{\text{work}}(v)}_{\text{Working}} \]

定理 1 — 最大研究深度

\[ d^* = \left\lfloor \frac{C - S - D}{\alpha \cdot \overline{\Delta I}} \right\rfloor \]

\(D\)：单次实验的上下文需求（\(c_{\text{code}} + c_{\text{env}} + c_{\text{cot}} + c_{\text{out}}\)）
\(\overline{\Delta I}\)：每次实验的信息增量（理想压缩后）
\(\alpha\)：实际压缩 / 理想压缩的比值

定理 2 — 可解性条件

\[ C_{\min} = S + D + k^* \cdot \alpha \cdot \overline{\Delta I} \]

\(k^*\)：解决问题所需的最少实验次数
\(C_{\min}\)：使问题可解的最小上下文窗口

关键洞察：压缩比 \(\alpha\) 才是主导因素

理想压缩 — 下界

"STGCN, h=64, lr=1e-3 → MAE=4.23"
≈ 50 tokens

实际 LLM 摘要

"We tested STGCN with hidden dim
64... converged at epoch 80..."
≈ 400–800 tokens

\(\alpha \approx 8 \sim 16\)

核心发现：由定理 1，实际研究深度 \(d^*_{\text{actual}} = d^*_{\text{ideal}} / \alpha\)。

\(\alpha\) 每增大一倍，可做的实验次数减半。

启示：提升 AI 科研能力的关键不只是加大上下文窗口，更在于优化上下文压缩策略——从 \(\alpha = 10\) 压缩到 \(\alpha = 2\) 的收益相当于把窗口从 128K 扩大到 640K。

本研究同时报告理想压缩（理论下界）和实际压缩（实测值），形成对比分析。

研究计划与预期贡献

研究计划

Phase 1 交通预测案例
实测各上下文分量 → 校准参数 → 深度极限测试 → 压缩比测量

Phase 2 跨问题泛化
扩展至网络均衡 + 信号优化，验证 \(C_{\min}\) 排序

Phase 3 工程优化建议
基于分析结果，指导压缩策略和 Agent 架构设计

预期贡献

1CCTS 理论框架 — 首次将 AI 自主科研建模为上下文约束树搜索

2可解性分析 — 基于 \(C_{\min}\) 定量评估哪些交通研究可交给 AI

3压缩差距量化 — 实测 \(\alpha\)，指明"优化压缩比扩大窗口更有效"

4考虑上下文约束的搜索算法 — 在有限上下文空间内搜索科研问题解法路径
