研究背景

AI Scientist 范式

基于 LLM 的 Agent（如 The AI Scientist、MLAgentBench）已能自主开展科学研究：生成想法、运行实验、撰写论文。

核心瓶颈

在处理复杂研究任务时，这些 Agent 频繁遇到 上下文窗口限制——LLM 单次能处理的最大 token 数。

为什么上下文对科研很重要？

科研 = 迭代试错。随着实验积累，历史信息不断压缩可用于当前任务的上下文空间。

核心问题：上下文空间被逐步压缩

随着研究推进（树深度增加），可用工作上下文持续缩小：

当 c_work < D（任务需求）时，Agent 无法完成下一次实验。

研究范围

完整的 AI 科研流程分为三个阶段，本研究聚焦第二阶段：

阶段一
Idea 生成
阶段二
自主实验
阶段三

核心问题：给定上下文窗口 \(C\)，AI Agent 能否通过迭代实验完成给定的交通工程研究任务？

不评估创新性和写作质量——仅关注任务完成能力。

研究问题与假设

给定 LLM 上下文窗口 \(C\)，哪些交通工程研究问题可以被 AI Agent 自主解决？
其可解性边界能否被理论预测？

H1 — 深度上界 Agent 能执行的连续实验次数存在可计算的上界 \(d^*\)，且接近 \(d^*\) 时性能显著下降。

H2 — 问题可分性 不同问题类型有不同的最小上下文需求 \(C_{\min}\)，据此可将问题分为"可解"与"不可解"。

H3 — 压缩差距 理想压缩与实际 LLM 摘要的差距（压缩比 \(\alpha\)）是限制实际研究深度的主导因素。

CCTS：上下文约束树搜索

Research Tree \(\mathcal{T} = (V, E, r)\)

关键定义

  • 节点 \(v\)：一次有具体结果的实验
  • 边 \((u,v)\)：基于 \(u\) 的结果决策执行新实验
  • 深度 \(d(v)\)：从根到 \(v\) 的实验次数
  • 成功：存在 \(v\) 使 metric\((v) \leq \epsilon\)

每个节点代表一次实验，如：
"STGCN, hidden=64, lr=0.001 → MAE=4.23"

上下文分配模型

\[ C = \underbrace{c_{\text{sys}} + c_{\text{prob}}(\mathcal{P})}_{S(\mathcal{P}):\ \text{Static Overhead}} + \underbrace{c_{\text{hist}}(v)}_{\text{Inherited Context}} + \underbrace{c_{\text{work}}(v)}_{\text{Working Context}} \]

分量含义性质
\(c_{\text{sys}}\)Agent 系统指令、工具描述、行为规范常数，与问题无关
\(c_{\text{prob}}(\mathcal{P})\)问题定义：目标、数据描述、约束与问题类型相关的常数
\(c_{\text{hist}}(v)\)所有祖先实验的压缩摘要随深度单调递增
\(c_{\text{work}}(v)\)执行当前实验的剩余上下文随深度单调递减

\[ c_{\text{work}}(v) = C - S(\mathcal{P}) - c_{\text{hist}}(v) \]

任务上下文需求

执行一次实验所需的最小上下文：

\[ D(\mathcal{P}) = c_{\text{code}} + c_{\text{env}} + c_{\text{cot}} + c_{\text{out}} \]

\(c_{\text{code}}\)

需要读写的代码
（数据加载、模型、训练、评估）

\(c_{\text{env}}\)

环境上下文
（API 文档、数据 schema、文件结构）

\(c_{\text{cot}}\)

推理过程
（规划、调试、决策）

\(c_{\text{out}}\)

实验输出
（训练日志、指标、报错信息）

可行性条件：节点 \(v\) 处实验可行 ⟺ \(c_{\text{work}}(v) \geq D(\mathcal{P})\)

压缩假设

理想压缩（下界）

将实验结果压缩为最短描述长度：

"STGCN, h=64, lr=1e-3
→ MAE=4.23, RMSE=6.15"
≈ 50 tokens

\(c_{\text{hist}}(v) = d \cdot \overline{\Delta I}\)

实际 LLM 摘要

LLM 摘要包含动机、分析和建议：

"We tested STGCN with hidden
dim 64 and lr 1e-3. The model
converged at epoch 80..."
≈ 400–800 tokens

\(c_{\text{hist}}^{\text{actual}} = \alpha \cdot d \cdot \overline{\Delta I}\)

压缩比: \(\alpha = \dfrac{c_{\text{hist}}^{\text{actual}}}{c_{\text{hist}}^{\text{ideal}}} \approx 8 \sim 16\)

核心结果：最大研究深度

定理 1（Maximum Research Depth）

\[ d^*(\mathcal{P}, C, \alpha) = \left\lfloor \frac{C - S(\mathcal{P}) - D(\mathcal{P})}{\alpha \cdot \overline{\Delta I}} \right\rfloor \]

\(C - S - D\)

可用于存储历史
的上下文预算

\(\alpha \cdot \overline{\Delta I}\)

每次实验记录
的上下文成本

\(d^*\)

最大连续
实验次数

直观理解：上下文越大、问题固有开销越小、压缩越好（\(\alpha\) 越小）→ 能做更多实验 → 成功概率越高。

可解性条件

设 \(k^*(\mathcal{P})\) = 解决问题 \(\mathcal{P}\) 所需的最少实验次数。

定理 2（Solvability Condition）

\[ d^*(\mathcal{P}, C, \alpha) \geq k^*(\mathcal{P}) \]

等价地，解决问题所需的最小上下文窗口：

\[ C_{\min} = \underbrace{S(\mathcal{P})}_{\text{"Entry Fee"}} + \underbrace{D(\mathcal{P})}_{\text{"Workbench"}} + \underbrace{k^* \cdot \alpha \cdot \overline{\Delta I}}_{\text{"Memory Cost"}} \]

实验设计 — 第一阶段

实验 1 参数校准 — 让 Agent 执行交通预测任务，记录各上下文分量的实际 token 数，与理论估算对比。

实验 2 深度极限测试 — 控制上下文窗口（32K / 64K / 128K），记录 Agent 在不同深度的实验成功率。验证 H1。

实验 3 压缩比测量 — 对比三种摘要方式（理想/LLM 摘要/结构化模板），测量实际 \(\alpha\) 值。验证 H3。

实验 4 难度梯度 — 设置三级难度：L1 单步 MLP → L2 多步 GCN → L3 架构搜索。估算各级 \(k^*\)。验证 H2。

预期贡献

1CCTS 理论框架 — 首次将 AI 自主科研建模为上下文约束树搜索问题，提供分析 AI Agent 能力边界的理论工具。

2可解性分析方法 — 基于 \(C_{\min}\) 定量评估哪些交通研究可以交给 AI 完成。

3压缩差距量化 — 首次实测理想压缩与 LLM 摘要的差距 \(\alpha\)，为工程优化指明方向。

4AI 可解性图谱 — 基于上下文需求对典型交通问题分类，为研究者提供实用参考。

局限性与未来方向

当前简化假设

  • 单路径模型：未建模跨分支的信息共享
  • 固定需求假设：假设 \(D(\mathcal{P})\) 对同类问题为常数
  • 忽略注意力退化："Lost in the Middle" 效应可能导致提前退化

未来方向

  • 机制 B 建模：引入上下文长度-准确率衰减函数
  • 全流程扩展：将 CCTS 扩展至 Idea 生成和论文撰写
  • 跨领域泛化：材料科学、药物发现等其他领域
  • 工程优化：动态压缩、选择性遗忘、外部记忆增强
